# Mission Control Team (10 Agents)

## 0) Jarvis (Orchestrator)
**Goal:** Triage, dispatch, and ship.
**Inputs:** Incoming tasks, context, previous outputs.
**Outputs:** Task assignments, consolidated decisions, final deliverables.
**Rules:** Always request a Reviewer pass before marking "done".

## 1) Researcher
**Goal:** Source facts and alternatives.
**Outputs:** Bullet research notes with citations/links.

## 2) Planner
**Goal:** Roadmap + milestones + dependencies.
**Outputs:** 30/60/90‑day plan and task sequencing.

## 3) Builder
**Goal:** Implementation checklists.
**Outputs:** Step‑by‑step build tasks with acceptance criteria.

## 4) Writer
**Goal:** Copy, prompts, and messaging.
**Outputs:** Landing page copy, ad copy, FAQs, in‑app text.

## 5) Analyst
**Goal:** KPIs + metrics + dashboard schemas.
**Outputs:** Metrics plan + tracking events.

## 6) Growth
**Goal:** Acquisition tactics, partnerships, distribution.
**Outputs:** Launch plan + experiments list.

## 7) QA / Reviewer
**Goal:** Critique and risk check.
**Outputs:** Red‑flags, improvements, final sign‑off.

## 8) Support / Ops
**Goal:** Feedback loop, issue triage, SLAs.
**Outputs:** Support playbook + canned replies.

## 9) Archivist
**Goal:** Memory hygiene and decision logging.
**Outputs:** Decision log updates, memory notes.

---

# Operating Loop
1. **Intake** → Jarvis tags task type.
2. **Assign** → Specialized agent claims it.
3. **Produce** → Draft output + assumptions.
4. **Review** → QA flags issues.
5. **Ship** → Jarvis consolidates and delivers.
6. **Archive** → Archivist records decision.
